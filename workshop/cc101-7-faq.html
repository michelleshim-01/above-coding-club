자주 묻는 질문

Claude Code 입문자가 자주 궁금해하는 질문과 답변.

Q1. Claude랑 Claude Code랑 무슨 차이에요?
  • Claude = Anthropic이 만든 LLM 모델 (AI 두뇌)
  • Claude Code = Claude 모델 + 다양한 도구(Tool)를 실행할 수 있는 AI 코딩 에이전트

Tool 예시: Edit(파일 수정), Web Search(웹 검색), Bash(명령어 실행) 등

즉, 코드를 생성하는 것뿐 아니라 실행까지 자동화하는 도구입니다.

ClaudeClaude Code
작동 위치웹 브라우저내 컴퓨터 터미널
할 수 있는 일대화, 질문 답변파일 생성/수정, 명령어 실행
비유카카오톡 상담사직접 출장 온 전문가
Q2. Claude Desktop이랑 Claude Code랑 무슨 차이에요?

발전 순서:

  1. Claude 모델 개발
  2. Claude 웹 (claude.ai) 개발
  3. Claude Desktop 개발 (채팅 탭)
  4. Claude Code 개발 (터미널 CLI)
  5. Claude Desktop에 Code 탭 추가
  6. Claude Desktop에 Cowork 탭 추가

핵심 차이:

  • Claude Code = 개발자를 위한 도구 (터미널 CLI)
  • Claude Desktop = 비개발자를 위한 도구 (채팅 / Cowork / Code 탭 통합)

뒤에서 돌아가는 Claude 모델은 동일합니다. 단, 시스템 프롬프트와 도구(Tool) 구성이 다르기 때문에 체감 성능은 다를 수 있습니다. Claude Desktop의 Code 탭은 Claude Code와 동일한데, GUI냐 CLI냐의 차이입니다.

Q3. 터미널은 뭐에요?

어원: 라틴어 Terminalis(끝, 종착점)에서 유래. 옛날 중앙 컴퓨터의 끝단(terminal point)에 연결된 입출력 장치를 터미널이라고 불렀습니다.

지금의 터미널: 버튼 없이 직접 명령을 보내서 실행하는 창. 컴퓨터와 소통하는 매개체입니다.

GUI터미널
조작 방식마우스로 클릭키보드로 타이핑
예시윈도우 탐색기, macOS FinderTerminal(Mac), PowerShell(Win)
비유터치스크린 키오스크직원에게 직접 말로 주문
💡 안심하세요
Claude Code를 쓸 때는 터미널 명령어를 외울 필요가 없습니다. 한국어로 하고 싶은 말을 입력하면 됩니다.
Q4. CLI는 뭐에요?

CLI = Command Line Interface (명령줄 인터페이스)

개발 환경에서는 CLI 기반으로 많이 개발합니다. CLI의 장점:

  • 반복 작업에 강함 — 파일 이름 100개 바꿔야 할 때, GUI는 우클릭 100번. CLI는 반복문 한 줄이면 끝
  • 조합이 자유로움 — GUI는 메뉴 버튼만 누를 수 있지만, CLI는 명령어를 자유롭게 조합 가능
  • 자동화 — 반복적인 명령어들을 스크립트(.sh, .bat)로 만들어 한 번에 실행 가능
Q5. IDE(VS Code)에서 쓰는 것과 터미널의 차이

IDE = Integrated Development Environment (통합 개발 환경)

VS Code 같은 것들이 IDE입니다. 코드를 확인할 수 있는 공간 + 개발 편의 도구가 함께 있습니다. IDE 안에도 터미널이 내장되어 있어요.

IDE (VS Code)터미널
역할코드를 보고 편집하는 작업 공간컴퓨터에게 명령을 내리는 창
비유워드 프로세서메모장 + 리모컨
Claude Code와의 관계VS Code 안에서 터미널을 열어서 사용Claude Code가 실행되는 곳

그럼 왜 터미널을 선호하는가?

Claude Code를 많이 사용하면서 이제 코드를 거의 보지 않는 사람들이 늘고 있습니다 → 터미널로 충분. 개발하는 방식의 차이일 뿐, 더 좋고 나쁘고의 문제가 아닙니다.

💡 비개발자에게는
VS Code 안에 터미널이 내장되어 있어서, VS Code 하나만 열면 둘 다 사용할 수 있습니다. 파일이 어떻게 바뀌는지 눈으로 확인할 수 있어서 처음에는 VS Code를 권장합니다.
