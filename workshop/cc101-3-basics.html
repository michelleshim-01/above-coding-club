기본 사용법

Permission Mode, 슬래시 명령어, 단축키를 익히고 첫 실습을 해봅니다.

첫 실행

1

작업 폴더 열기

바탕화면의 my-workshop 폴더를 VS Code로 열기 (File → Open Folder)

2

Claude Code 실행

VS Code 하단 터미널 열기 (Ctrl+`) → claude 입력 → 테마 선택 (Light 권장)

3

첫 대화

안녕하세요! 이 폴더에 "hello.txt" 파일을 만들어줘. 내용은 "Claude Code 워크샵 시작!" 입력 후 Enter

💡 터미널 = 대화창
터미널이 무섭게 느껴질 수 있지만, 그냥 Claude와 대화하는 창이라고 생각하세요. 명령어를 외울 필요가 없습니다. 하고 싶은 걸 말로 입력하면 됩니다.

Permission Mode

Shift+Tab을 누르면 default → acceptEdits → plan 순서로 순환됩니다. (auto·bypassPermissions는 별도 활성화 후에만 순환에 포함)

모드자동 승인되는 것언제 쓸까?
default (Ask before edits)읽기만✅ 처음 시작할 때 (권장)
acceptEdits (Edit automatically)읽기 + 파일 편집 + 안전한 파일시스템 명령어 (mkdir, touch, mv, cp, rm, sed 등)코드 변경을 나중에 git diff로 한꺼번에 검토하고 싶을 때
plan (Plan mode)읽기만 (탐색용 셸 명령은 매번 확인). 파일 수정 안 함 — 계획만 작성작업 전 방향 확인할 때
auto (Auto mode) — Max·Team·Enterprise·API 한정대부분 — 별도 분류 모델이 위험한 액션(외부 전송, 프로덕션 배포, 강제 push 등)만 차단긴 작업, 개입 줄이고 싶을 때 (Pro 플랜 미지원)
⚠️ bypassPermissions (Bypass)거의 모든 것 (.git·.claude.json 같은 보호 경로만 예외)샌드박스/컨테이너 등 격리 환경에서만
⚠️ bypassPermissions 모드 주의
이 모드는 Claude가 거의 모든 명령을 사전 승인 없이 실행합니다. 평상시 Mac/Windows 노트북에서는 쓰지 마세요. Default/Plan 모드로 충분히 익숙해진 후, Git 백업이 있을 때 컨테이너·VM 같은 격리 환경에서만 사용하는 게 안전합니다. Anthropic은 일반 환경에선 대신 auto 모드(별도 분류 모델이 위험 차단)를 권장해요 — 단, Pro 플랜에선 사용 불가입니다.

Plan Mode 실습

Plan Mode는 Claude가 파일을 수정하지 않고 코드를 읽고 탐색해서 계획만 세웁니다. (탐색용 셸 명령은 일반 모드처럼 매번 확인을 거쳐 실행됨)

Claude Code 입력
내 폴더에 간단한 메모 앱 HTML 파일을 만들려고 해.
어떤 파일들이 필요하고 어떻게 만들 건지 계획 세워줘. 지금은 만들지 말고.

→ 계획이 마음에 들면 모드 선택 화면에서 "승인하고 실행" 옵션을 고르거나, Shift+Tab으로 default 모드로 빠져나가서 실행 요청.

핵심 슬래시 명령어

프롬프트 입력창에서 /를 입력하면 목록이 나옵니다.

명령어역할언제?
/init현재 폴더를 분석해서 CLAUDE.md 자동 생성새 프로젝트 시작 시
/planPlan Mode 진입 (파일 수정 없이 계획 작성)큰 작업 전 검토
/clear새 대화 시작 (이전 컨텍스트 비우기). 이전 대화는 /resume으로 다시 열 수 있음하나의 작업 완료 후
/compact현재 대화를 요약해서 컨텍스트 공간 확보대화가 길어질 때
/context현재 컨텍스트 사용량을 시각화대화가 무거워졌나 확인할 때
/resume이전 세션 불러오기 (별칭: /continue)/clear 후 돌아가고 싶을 때
/usage세션 비용·플랜 사용량·통계 확인 (별칭: /cost, /stats)사용량 모니터링
/memoryCLAUDE.md 편집 + 자동 기억(auto-memory) 확인/관리Claude가 뭘 기억하는지 볼 때
/model사용 모델 변경다른 모델을 시도하고 싶을 때
/fastFast 모드 켜고 끄기응답 속도를 높이고 싶을 때
/help모든 명령어 목록 보기언제든지
📊 컨텍스트 관리 원칙
AI 모델은 대화를 "기억"하는 게 아니라 모든 대화 내용을 매번 다시 읽습니다. 대화가 길어질수록 느려지고 비용이 늘어납니다.

Anthropic 엔지니어들의 권장 패턴:
• 질문-답변 5쌍 이내에 한 작업 단위 완료
• 작업 완료 → /clear로 새 세션 시작
• 컨텍스트 50% 남았을 때 → /compact으로 미리 압축

키보드 단축키

Shift+Tab 모드 전환
Shift+Enter 줄바꿈 (Enter는 전송)
Ctrl+C 작업 중단
Esc 현재 입력 취소
EscEsc 직전 작업 취소 (되돌리기)
Alt+P 모델 전환
Alt+T 사고 모드 전환
!명령어 터미널 Bash 직접 실행
⚠️ Esc × 2 주의사항
Esc를 두 번 누르면 Claude가 수정한 파일을 되돌립니다. 단, 사용자가 직접 수정한 내용은 되돌아가지 않습니다. Claude 작업만 취소됩니다.

! 로 터미널 명령어 실행

Claude Code 입력창에서 !로 시작하면 터미널 명령어를 직접 실행합니다.

예시
!ls           # 현재 폴더 파일 목록
!node app.js  # 파일 실행
!git status   # Git 상태 확인
← 이전 VS Code 활용법 다음 → Git 안전망
